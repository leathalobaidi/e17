Recommended Tech Stack

Optimised for low cost, high performance, and minimal maintenance. Primary picks are battle-tested and free where possible.

CF
Hosting
Cloudflare Pages Primary
Free tier with unlimited bandwidth, global CDN, instant deploys from Git. Fastest option for static sites with automatic SSL and preview deployments.
Alternatives: Netlify (free tier, 100GB/mo) • DigitalOcean Droplet ($5/mo VPS for dynamic sites)
D
Domain
Namecheap / Cloudflare Registrar
Cloudflare Registrar sells domains at wholesale cost (no markup). Namecheap offers competitive pricing with good WHOIS privacy.
Typical cost: .co.uk ~$8/yr • .com ~$10-12/yr
@
Email
Zoho Mail / Google Workspace
Zoho Mail free tier gives 1 user, 5GB storage with custom domain. Google Workspace at $5/mo provides full Gmail, Drive, and calendar.
Budget option: Cloudflare Email Routing (free forwarding to Gmail)
DNS
DNS
Cloudflare DNS
Free managed DNS with DDoS protection, fast propagation, and built-in analytics. Industry-leading performance and reliability.
Bonus: Proxy mode adds CDN + security layer at no cost
SSL
SSL
Let's Encrypt (auto)
Free, automatic SSL via Cloudflare Pages or Netlify. Zero configuration needed — certificates auto-renew every 90 days.
For VPS: Use Certbot for automatic Let's Encrypt provisioning
A
Analytics
Plausible / Google Analytics
Plausible ($9/mo) is privacy-friendly, lightweight (< 1KB), and GDPR-compliant with no cookie banner needed. GA4 is free but heavier.
Free alternative: Cloudflare Web Analytics (basic, privacy-first)
F
Forms
Formspree / Netlify Forms
Formspree free tier handles 50 submissions/month with spam filtering. Netlify Forms is built-in if you host on Netlify (100 submissions/mo).
Self-hosted: Simple mailto: link or custom worker function
M
Maps
Google Maps Embed
Free, no API key needed. Embed a map iframe directly from Google Maps. Sufficient for showing a business location with directions link.
Alternative: OpenStreetMap embed (fully free, no Google dependency)

Common Troubleshooting

Quick fixes for the most frequent deployment issues. Click to expand each solution.

!
DNS propagation delay — site not loading after domain change
▶
Cause: DNS changes can take 5 minutes to 48 hours to propagate globally.
  • Use dig businessname.co.uk or whatsmydns.net to check propagation status
  • Flush local DNS cache: sudo dscacheutil -flushcache (macOS) or ipconfig /flushdns (Windows)
  • If using Cloudflare, propagation is typically under 5 minutes with proxied records
  • Reduce TTL to 300 (5 min) before making DNS changes, then increase after propagation
Quick fix: Use Cloudflare as your nameserver for near-instant propagation.
!
SSL mixed content — padlock missing or security warnings
▶
Cause: HTTP resources (images, scripts, fonts) loaded on an HTTPS page.
  • Open browser DevTools > Console and look for "Mixed Content" warnings
  • Find and replace all http:// URLs with https:// or use protocol-relative //
  • Add <meta http-equiv="Content-Security-Policy" content="upgrade-insecure-requests"> as a quick fix
  • Check third-party embeds (maps, fonts, widgets) are loaded over HTTPS
Prevention: Always use relative paths for local assets and https:// for externals.
!
Google Maps iframe blocked — embed not displaying
▶
Cause: Content Security Policy (CSP) headers or X-Frame-Options blocking the iframe.
  • Ensure your CSP header includes frame-src https://www.google.com
  • Use the official Google Maps embed URL format: https://www.google.com/maps/embed?pb=...
  • Do NOT use the shortened goo.gl/maps links — they redirect and may be blocked
  • Test the iframe in an incognito window to rule out browser extensions
Tip: Get the embed code directly from Google Maps > Share > Embed a map.
!
Mobile viewport issues — content overflowing or too small
▶
Cause: Missing viewport meta tag or fixed-width elements.
  • Ensure <meta name="viewport" content="width=device-width, initial-scale=1.0"> is in the <head>
  • Avoid fixed pixel widths on containers — use max-width and percentages
  • Check for horizontal scrollbars with overflow-x: hidden on the body (temporary debug)
  • Test on real devices, not just DevTools — touch targets need minimum 44px height
Debug tip: Add * { outline: 1px solid red; } to CSS to find overflowing elements.
!
Slow page loading — PageSpeed score below 90
▶
Cause: Large images, render-blocking scripts, or unoptimised assets.
  • Compress images with Squoosh or TinyPNG — use WebP format where possible
  • Add loading="lazy" to images below the fold
  • Move non-critical JS to <script defer> or end of <body>
  • Inline critical CSS and defer non-essential stylesheets
  • Enable Cloudflare's auto-minify for HTML, CSS, and JS
  • Use font-display: swap on custom fonts to prevent invisible text
Target: LCP < 2.5s, FID < 100ms, CLS < 0.1 (Core Web Vitals).

Post-Launch Timeline

What to do after the site goes live. Follow this timeline for every new client deployment.

Day 1
Submit to Google Search Console
Add the site property, verify ownership via DNS TXT record, and submit the sitemap.xml. Request indexing for the homepage and key landing pages.
Week 1
Share with client and gather feedback
Send the live URL to the client. Walk them through the site on a call. Collect any text corrections, image swaps, or layout adjustments needed.
Week 2
First Google indexing check
Verify pages are appearing in Google Search Console's coverage report. Check for crawl errors, excluded pages, or indexing issues. Fix any warnings.
Month 1
First analytics report
Pull the first traffic report: visitors, page views, bounce rate, top pages, device breakdown. Share a summary with the client showing early traction.
Month 2
SEO rankings check
Check Search Console for keyword impressions and positions. Identify the top queries driving traffic and any quick-win keywords to optimise for.
Month 3
First quarterly review
Comprehensive review: traffic growth, conversion rates, SEO progress, site speed, and client satisfaction. Set goals for the next quarter. Discuss upsell opportunities.
